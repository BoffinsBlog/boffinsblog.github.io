Steps#

I knew this process was going to be awkward but I didn’t expect it to consume so much of my time.

Everywhere I read led me to manually install and run an acme.sh script.

That I could not get to work.

However with a combination of luci-app-acme and Terminal commands* I finally got there.

(* I may be doing this all wrong but I think that luci-app-acme doesn’t write some config settings correctly; hence the Terminal to amend this and then run the initial certificate generation.)

Install#

The SSL module is not installed by default:

opkg update

opkg install lighttpd-mod-openssl

Add the acme modules and the LuCI bits:

opkg install acme luci-app-acme

The symlink that acme created in the .well-known directory is wrong, recreate:

rm -R /www/.well-known/

mkdir /www/.well-known/

ln -s /var/run/acme/challenge/ /www/.well-known/acme-challenge

Enable the acme-challenge:

nano /etc/lighttpd/lighttpd.conf

# ACME-challenge
$HTTP["url"] =~ "^/[.]well-known/acme-challenge/" {
    alias.url += ( "/.well-known/acme-challenge" => "/www/.well-known/acme-challenge/" )
}

Restart:

service lighttpd restart

service lighttpd status

Check that the acme-challenge is working as intended:

nano /var/run/acme/challenge/README.txt

(Insert some text here, like ‘Oh, hello.’.)

In a different Terminal:

curl -L -v http://<DOMAIN_NAME>/.well-known/acme-challenge/README.txt

In LuCI:

http://<PLUG_IP>/luci

Services | ACME certificates

Save & Apply

Restart the web server and acme processes:

service lighttpd restart

/etc/init.d/acme restart

At this point it might be wise to cast an eye over these files, especially the e-mail address:

nano /etc/config/acme

nano /etc/acme/ca/acme-v02.api.letsencrypt.org/directory/ca.conf

nano /etc/acme/<DOMAIN_NAME>/<DOMAIN_NAME>.conf

Restart the server and acme process if any changes are made.

Trigger a renewal process:

/etc/init.d/acme renew

Note: If a certificate has previously been generated, using the same details, then that certificate will be retrieved and a new one will not be created.

Result:

Cert success.
Your cert is in: /etc/acme/<DOMAIN_NAME>/<DOMAIN_NAME>.cer
Your cert key is in: /etc/acme/<DOMAIN_NAME>/<DOMAIN_NAME>.key
The intermediate CA cert is in: /etc/acme/<DOMAIN_NAME>/ca.cer
And the full-chain cert is in: /etc/acme/<DOMAIN_NAME>/fullchain.cer
Running renew hook: '/usr/lib/acme/notify renewed'

Add the newly created certificate to the vHost configuration.

nano /etc/lighttpd/vhosts.d/<DOMAIN_NAME>.conf

Full working example below:

# Site <DOMAIN_NAME>
$HTTP["host"] == "<DOMAIN_NAME>" {
    server.document-root = "/www/sites/<DOMAIN_NAME>"
    server.errorlog = "/var/log/lighttpd/<DOMAIN_NAME>.error.log"
    accesslog.filename = "/var/log/lighttpd/<DOMAIN_NAME>.access.log"

    $SERVER["socket"] == ":443" {
        ssl.engine = "enable"
        ssl.pemfile = "/etc/acme/<DOMAIN_NAME>/combined.cer"
    }
}

Restart the web server and acme processes:

service lighttpd restart

/etc/init.d/acme restart

crontab#

Run the renew process daily at midnight to update any certs as needed:

crontab -e

# Renew the acme certs daily at midnight
0 0 * * * /etc/init.d/acme renew

Redirecting from HTTP to HTTPS#

Add mod_redirect.

nano /etc/lighttpd/lighttpd.conf

# Added by me
# Modules
server.modules += (
        "mod_alias",
        "mod_redirect"
)

Add the redirect to the vHosts.

nano /etc/lighttpd/vhosts.d/<DOMAIN_NAME>.conf

Full working example below:

# Site <DOMAIN_NAME>
$HTTP["host"] == "<DOMAIN_NAME>" {
    server.document-root = "/www/sites/<DOMAIN_NAME>"
    server.errorlog = "/var/log/lighttpd/<DOMAIN_NAME>.error.log"
    accesslog.filename = "/var/log/lighttpd/<DOMAIN_NAME>.access.log"

    $SERVER["socket"] == ":443" {
        ssl.engine = "enable"
        ssl.pemfile = "/etc/acme/<DOMAIN_NAME>/combined.cer"
    }

    $SERVER["socket"] == ":80" {
        $HTTP["host"] =~ "(.*)" {
                url.redirect = ( "^/(.*)" => "https://%1/$1" )
        }
    }
}

Conclusion#

# BEWARE #

Too many unsuccessful certificate creation attempts will result in being locked out for an hour.

https://letsencrypt.org/docs/rate-limits/#authorization-failures-per-identifier-per-account

Too many successful (5) certificate creations will result in a lockout of a day.

https://letsencrypt.org/docs/rate-limits/#new-certificates-per-exact-set-of-identifiers

So, don’t do what I did and generate a working cert, then delete / uninstall everything and repeatedly try again to ensure the documented steps were correct.

Notes (stuff I found out later)#

Removing acme:

service acme stop

opkg remove acme* --force-depends

opkg remove luci-app-acme

opkg remove lighttpd-mod-openssl

rm -R /etc/acme*

rm /etc/config/acme

Ensure any rows regarding $SERVER[“socket”] == “:443” are either removed or commented out in:

nano /etc/lighttpd/vhosts.d/<DOMAIN_NAME>.conf

service lighttpd restart

service lighttpd status
