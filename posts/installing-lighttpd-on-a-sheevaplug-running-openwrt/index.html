Files and Tools Needed#

As I was tinkering once again, ruminating on how to use my SheevaPlug as a web server*, I thought I’d put some notes together around the process.

* This is actually something I’ve been thinking about for a while, for a couple of reasons:

  1. A separate dedicated web server is never a bad idea

  2. It means that I can take the main server offline and perform updates without bringing my websites down

Resources#

https://openwrt.org/docs/guide-user/services/webserver/lighttpd

https://openwrt.org/docs/guide-user/luci/luci.on.lighttpd

Steps#

Stop uhttpd#

service uhttpd stop

Install lighttpd#

Install lighttpd and the LuCI packages:

su

opkg update

opkg install lighttpd lighttpd-mod-cgi luci-mod-admin-full

Add a LuCI theme:

opkg install luci-theme-openwrt

Add some optional modules.

For logging:

opkg install lighttpd-mod-accesslog

Start lighttpd#

service lighttpd start

service lighttpd status

Create some directories#

Content needs a place to live.

I put all my content under /www/sites/:

mkdir -p /www/sites/sheevaplug/
mkdir /www/sites/sheevaplug/blog/
mkdir /www/sites/sheevaplug/code/

Configure#

Configure the landing pages#

This is the default landing page; the page that will be served when typing the ‘Plug IP into a browser.

Check that the cgi-module has been loaded:

nano /etc/lighttpd/conf.d/30-cgi.conf

Ensure that this line is there:

server.modules += ( "mod_cgi" )

In the same file, tell lighttpd to process LuCI CGI requests.

I always keep original comments and add to them; that way I can revert to a known good prior state easily:

# Changed by me
#cgi.assign                 = ( ".pl"  => "/usr/bin/perl",
#                               ".cgi" => "/usr/bin/perl",
#                               ".rb"  => "/usr/bin/ruby",
#                               ".erb" => "/usr/bin/eruby",
#                               ".py"  => "/usr/bin/python" )

# Added by me
cgi.assign                 = ( ".pl"  => "/usr/bin/perl",
                                ".cgi" => "/usr/bin/perl",
                                ".rb"  => "/usr/bin/ruby",
                                ".erb" => "/usr/bin/eruby",
                                ".py"  => "/usr/bin/python",
                                "/cgi-bin/luci" => "",
                                "/cgi-bin/cgi-backup" => "",
                                "/cgi-bin/cgi-download" => "",
                                "/cgi-bin/cgi-exec" => "",
                                "/cgi-bin/cgi-upload" => "",
                                )

LuCI needs root privileges to access configuration, so lighttpd needs to run as root too:

nano /etc/lighttpd/lighttpd.conf

# Changed by me
#server.username             = "http"
#server.groupname            = "www-data"

At the bottom:

# Added by me
# Modules
server.modules += (
        "mod_alias"
)

Check and restart#

Check the configuration syntax:

lighttpd -t -f /etc/lighttpd/lighttpd.conf

Restart:

service lighttpd restart

Check the status:

service lighttpd status

Add vHosts#

As with nginx I am serving several sites on my ‘Plug so will use vHosts.

Create a home for the conf files:

mkdir -p /etc/lighttpd/vhosts.d/

nano /etc/lighttpd/vhosts.d/<DOMAIN_NAME>.conf

# Site <DOMAIN_NAME>
$HTTP["host"] == "<DOMAIN_NAME>" {
    server.document-root = "/www/sites/<DOMAIN_NAME>"
    server.errorlog = "/var/log/lighttpd/<DOMAIN_NAME>.error.log"
    accesslog.filename = "/var/log/lighttpd/<DOMAIN_NAME>.access.log"
}

Load the vHosts#

nano /etc/lighttpd/lighttpd.conf

# Include the vHosts
include_shell "cat /etc/lighttpd/vhosts.d/*.conf"

Check and restart

Add PHP#

opkg update
opkg install php8
opkg install php8-cgi
opkg install php8-mod-session

Edit / update:

nano /etc/lighttpd/lighttpd.conf

Change (comment out):

static-file.exclude-extensions = ( ".php", ".pl", ".fcgi" )

To:

# Changed by me
#static-file.exclude-extensions = ( ".php", ".pl", ".fcgi" )

Edit / update:

nano /etc/php.ini

Change (comment out):

doc_root = "/www"

To:

; Changed by me
;doc_root = "/www"

Edit / update:

nano /etc/lighttpd/conf.d/30-cgi.conf

# Added by me
cgi.assign                 = ( ".pl"  => "/usr/bin/perl",
                               ".cgi" => "/usr/bin/perl",
                               ".rb"  => "/usr/bin/ruby",
                               ".erb" => "/usr/bin/eruby",
                               ".py"  => "/usr/bin/python",
                               ".php" => "/usr/bin/php-cgi",
                               "/cgi-bin/luci" => "",
                               "/cgi-bin/cgi-backup" => "",
                               "/cgi-bin/cgi-download" => "",
                               "/cgi-bin/cgi-exec" => "",
                               "/cgi-bin/cgi-upload" => "",
                                )

Check and restart

Move LuCI#

This took far longer and more digging than I anticipated.

I don’t want the ‘Plug homepage to be the LuCI logon page.

(Yes, the easiest way would be to rename /www/index.html to something like /www/luci.html but what seemed like a simple change was proving to be more challenging and that was annoying me.)

I wanted http://<PLUG_IP>/luci to be the LuCI logon page.

Create the new LuCI home directory:

mkdir /www/luci/

Move the existing file:

mv /www/index.html /www/luci/index.html

Retain a copy for when I break it:

cp /www/luci/index.html /www/luci/index.html.old

The original LuCI index.html file can be found here:

https://github.com/openwrt/luci/blob/master/modules/luci-base/root/www/index.html

Edit:

nano /www/luci/index.html

Change:

<meta http-equiv="refresh" content="0; URL=cgi-bin/luci/" />

To:

<meta http-equiv="refresh" content="0; URL=/cgi-bin/luci/" />

Change:

<a href="cgi-bin/luci/">LuCI - Lua Configuration Interface</a>

To:

<a href="/cgi-bin/luci/">LuCI - Lua Configuration Interface</a>

Some of the LuCI controller configurations are hard coded so need changing manually:

nano /usr/lib/opkg/info/luci-base.list

Change:

/www/index.html

To:

/www/luci/index.html

Create a new /www/index.html file:

nano /www/index.html

The default lighttpd welcome page can be found here:

https://raw.githubusercontent.com/lighttpd/lighttpd2/refs/heads/master/contrib/default.html

Check and restart

Restricting LuCI#

To restrict which IP addresses can now access the http://<PLUG_IP>/luci directory:

nano /etc/lighttpd/lighttpd.conf

Add:

# Restrict access to /luci
$HTTP["remoteip"] !~ "192.168.0.xx|192.168.0.yy" {
    $HTTP["url"] =~ "^/luci/" {
        url.access-deny = ( "" )
    }
}

Check and restart

Hardening (or not as the case may be)#

The ‘Plug has one job: a web server.

By default the ‘Plug firewall only allows traffic through port 22 (SSH), port 80 (HTTP) and port 443 (HTTPS).

The ‘Plug sits behind a router which only has ports 80 and 443 open (the web traffic ports).

From outside the LAN the ‘Plug looks like this:

nmap -F <MY_FIXED_IP>
PORT    STATE SERVICE
80/tcp  open  http
443/tcp open  https

From inside the LAN, the ‘Plug looks like this:

nmap -F <SHEEVAPLUG_IP>
PORT     STATE SERVICE
22/tcp   open  ssh
53/tcp   open  domain
80/tcp   open  http

dnsmasq#

Port 53/tcp was being used by dnsmasq, which I stopped and disabled as I have no need of it:

/etc/init.d/dnsmasq stop

/etc/init.d/dnsmasq disable

/etc/init.d/dnsmasq status

Result!

nmap -F <SHEEVAPLUG_IP>
PORT    STATE SERVICE
22/tcp  open  ssh
80/tcp  open  http

Permissions#

Take ownership of all of the websites under /www/sites/:

chown http:www-data -R /www/sites/

Or if you prefer (depending on rights):

chown foo:foo -R /www/sites/

Set file and folder permissions for everything under /www/sites/:

find /www/sites/ -type f -exec chmod 644 {} + 
find /www/sites/ -type d -exec chmod 755 {} +

server.tag#

OK, this is juvenile but it made me smile; changing the server’s response header:

nano /etc/lighttpd/lighttpd.conf

# Change the server type if queried
server.tag = "nunyaBidniss v1.0"

Check and restart

Now have a look at:

curl -I http://<DOMAIN_NAME> | grep -i '^server:'

Remove uhttpd#

Remove the now redundant uhttpd:

opkg remove uhttpd --force-removal-of-dependent-packages

And any orphaned config file:

rm /etc/config/uhttpd

Conclusion#

OpenWrt does a spectacular job of running lighttpd and hosting a couple of simple sites.

Nothing fancy; flat HTML, CSS and some PHP.

Time to add SSL to the mix.

Notes (stuff I found out later)#

To see the output of all the config files parsed together:

lighttpd -p -f /etc/lighttpd/lighttpd.conf
